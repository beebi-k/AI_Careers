## Data Flow Diagram

### User Registration Flow
```
1. User enters credentials
   â†“
2. Frontend validates input
   â†“
3. Frontend calls authAPI.register()
   â†“
4. API sends POST /api/auth/register with data
   â†“
5. Backend receives request
   â†“
6. Backend validates email not duplicate
   â†“
7. Backend hashes password with bcrypt
   â†“
8. Backend creates user in MongoDB
   â†“
9. Backend generates JWT token
   â†“
10. Backend returns user + token
   â†“
11. Frontend stores token in localStorage
   â†“
12. Frontend stores user in Zustand store
   â†“
13. Frontend redirects to dashboard
```

### API Request with Authentication
```
1. User logged in (token in localStorage)
   â†“
2. Frontend calls resumeAPI.getAll()
   â†“
3. API client gets token from localStorage
   â†“
4. API client adds "Authorization: Bearer <token>" header
   â†“
5. Frontend sends GET /api/resumes
   â†“
6. Backend middleware receives request
   â†“
7. Middleware extracts token from header
   â†“
8. Middleware verifies token with JWT
   â†“
9. If valid, middleware sets req.userId
   â†“
10. Route handler (resumeController) gets req.userId
    â†“
11. Controller queries: Resume.find({ userId: req.userId })
    â†“
12. MongoDB returns user's resumes
    â†“
13. Controller returns resumes in response
    â†“
14. Frontend receives resumes
    â†“
15. Frontend renders resumes
```
